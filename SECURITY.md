# Security Policy

## Supported Scope

This repository is a public showcase. It may include architecture notes and selected public-safe assets.
Do not assume it contains full production infrastructure details.

## Reporting a Vulnerability

Please do not open public issues for suspected vulnerabilities.

Report privately via:

- GitHub security advisory (preferred)
- Email: pinopingpong@gmail.com

Include:

1. Affected area
2. Reproduction steps
3. Impact assessment
4. Suggested mitigation (optional)

I will acknowledge reports as soon as possible and provide remediation status updates.

## Sensitive Data Handling

- Do not commit credentials, tokens, OAuth secrets, or private keys.
- If accidental exposure occurs, rotate credentials first, then clean repository history if necessary.
