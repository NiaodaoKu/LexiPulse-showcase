# Contributing Guide

Thanks for your interest in improving LexiPulse Showcase.

## What Contributions Are Welcome

- Documentation clarity improvements
- Architecture explanations
- Demo polish suggestions
- Bug reports on public demo behavior

## Before Opening a PR

1. Open an issue for non-trivial changes.
2. Keep changes focused and small.
3. Explain rationale and expected impact.

## Pull Request Checklist

- [ ] Scope is clear and limited
- [ ] README/docs remain consistent
- [ ] No secrets or private environment values included
- [ ] Screenshots added when UI text/layout changes

## Commit Style

Use concise conventional-style prefixes when possible:

- feat: new capability
- fix: bug fix
- docs: documentation updates
- chore: maintenance changes

## Code of Conduct

Please communicate respectfully and constructively.
